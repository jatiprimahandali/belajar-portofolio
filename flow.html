Flow To-Do List

Diagram alur aplikasi To-Do List, dari load data sampai tiap aksi user.

Alur Aplikasi

flowchart TD
    A[Buka todo.html] --> B["loadTodos()
SELECT * FROM todos"]
    B -- error --> B1[Tampilkan pesan error]
    B -- sukses --> C[Simpan hasil ke array todos]
    C --> D["refreshKategoriOptions()"]
    D --> E["renderTable()"]

    E --> F{todos kosong?}
    F -- ya --> F1[Tampilkan emptyState]
    F -- tidak --> G["Hitung stats: total, selesai, pending, prioritas tinggi"]
    G --> H["getFiltered()
search + prioritas + kategori + status"]
    H --> I{hasil filter kosong?}
    I -- ya --> I1[Tampilkan noResultState]
    I -- tidak --> J["Sort: pending dulu → deadline terdekat/overdue → prioritas → created_at"]
    J --> K[Render baris tabel]

    K --> L1[User submit form Tambah Tugas]
    L1 --> L2["INSERT todos
task, kategori, prioritas, deadline"]
    L2 -- sukses --> L3["unshift(data) → renderTable()"]
    L2 -- error --> L4[Tampilkan formError + toast error]

    K --> M1[User klik checkbox Done]
    M1 --> M2["UPDATE is_done"]
    M2 -- sukses --> M3["update state → renderTable()"]
    M2 -- error --> M4[Kembalikan checkbox + toast error]

    K --> N1[User klik teks tugas]
    N1 --> N2[Tampilkan input inline edit]
    N2 --> N3["Simpan → UPDATE task"]
    N3 -- sukses --> N4["update state → renderTable()"]
    N3 -- error --> N5[Toast error, tetap di mode edit]

    K --> O1[User klik hapus]
    O1 --> O2{Konfirmasi hapus?}
    O2 -- batal --> K
    O2 -- ya --> O3["DELETE FROM todos"]
    O3 -- sukses --> O4["hapus dari array → renderTable()"]
    O3 -- error --> O5[Toast error]

    K --> P1[User ubah search / filter / reset]
    P1 --> H
Diagram ini disinkronkan manual setiap kali logika di todo.html berubah — bagian dari proses commit yang sama, bukan otomatis lewat CI.
